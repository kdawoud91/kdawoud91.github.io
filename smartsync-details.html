SmartSync: AIoT‑Enhanced Virtual Office Interactivity

Khaled Dawoud; Abdulmotaleb El Saddik — International Conference on Artificial Intelligence of Things (ICAIoT), 2024

AIoTHCIZoom APIsRealtime

Abstract (plain‑English)

Remote work pushed people to turn homes into offices. SmartSync connects virtual meeting platforms with the physical workspace using AIoT. The system reads meeting state (e.g., start, participants joined) and automates office peripherals—lighting, fan, and RGB indicators—while a lightweight agent notifies the host (emails, sounds) about key events. The goal is simple: make meetings smoother, more comfortable, and less manual for the user.

Motivation

The shift to remote work has created a clear need for workspaces that adapt automatically to virtual meetings. While platforms like Zoom evolved rapidly in software features, the physical environment remains disconnected. AIoT closes this gap by combining AI decision‑making with IoT sensing and actuation so rooms respond to meeting events. The result is improved comfort, fewer manual adjustments, and smoother collaboration—addressing a key shortcoming in current remote‑work setups.

Key Contributions

  • Automatic control of office peripherals (lamp, fan, RGB LEDs) driven by meeting lifecycle.
  • Lightweight agent for notifications and attendance awareness (welcome/reminder emails, "all present" signal).
  • End‑to‑end prototype linking a web‑conferencing API to physical devices in real time.

System Architecture

The prototype integrates a microcontroller with relays to switch AC devices, WS2812B LEDs for visual status, and a Python service that listens to the meeting platform and issues commands over serial.

Hardware
  • Microcontroller (Arduino‑class)
  • Relays for AC lamp & fan
  • WS2812B RGB LED strip (5V)
  • Dedicated 5V/20A PSU for LEDs
Software
  • Python service (PySerial) to drive hardware
  • Zoom APIs for meeting status & participants
  • SMTP for welcome/reminder emails
  • Optional TTS or audio prompts

Algorithms

Hardware control loop 
On defaults: lamp=OFF, fan=OFF, LED=GREEN
When meeting starts: lamp=ON, fan=ON, LED=RED
When all participants join: LED=BLUE, play notification sound
When meeting ends: restore defaults
Agent behaviour 
On meeting start: send welcome email with join link
If a specific participant is late: send reminder email
On participant join: optional chime
On all joined: notify host (sound) that meeting can begin

Qualitative Results

  • Automated peripheral control removes manual setup at meeting start/end.
  • RGB LEDs communicate meeting state (privacy/ready signals) at a glance.
  • Welcome + reminder emails improve punctuality and reduce host overhead.
  • Comfort factors (lighting/airflow) respond to the session state.

Limitations & Future Work

  • Prototype wiring; a wireless design (e.g., BLE/Wi‑Fi modules) suits real rooms better.
  • Educational‑grade components; smaller/cheaper boards (e.g., Nano‑class) are sufficient.
  • Planned: add vision cues (e.g., detect distraction/sleep) to trigger helpful nudges.
